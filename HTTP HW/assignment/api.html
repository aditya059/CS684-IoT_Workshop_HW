IoT HTTP Server API

General notes

IoT Server API

ReST API for communicating with IoT HTTP Server.

Overview

As part of the assignment, participants have to implement the given API in Python using requests library. While all the resources are usable on anything that supports HTTP, the ones for updating telemetry and receiving RPC commands are to be used on internet connected things or physical devices.

Authentication

There are two authentication schemes, one for authenticating the user and another for authenticating the thing which would be typically your micro-controllers and micro-computers.

The /login endpoint is used for authenticating the user. The returned authentication token should be sent in Authorization header as Bearer <token> for organization and management purposes such as creating and updating things, sending RPC commands. Unless otherwise specified, use this method for authentication of the resource.

The /thing/:id/token endpoint is used for authenticating the device. The returned access token should be copied to the device and sent in Authorization header as Bearer <token>.

Error Codes

2XX is returned for successful operation, with 200 returned most commonly. 4XX is used for client errors such as 401 for unauthorized user, 403 for wrong login credentials or invalid access, 422 for validation errors, etc.

Rate limit

Maximum of 10 requests per second. Extra requests will result in 503 service unavailable.
